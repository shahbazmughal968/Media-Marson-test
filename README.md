## API integration

- **RTK‑Query** is used for all data fetching.  The `baseAPI` slice (`src/services/baseAPI.js`) creates a `createApi` instance with a `fetchBaseQuery` pointing at the backend URL defined in `process.env.NEXT_PUBLIC_API_URL`.
- Individual endpoints are generated with `builder.query` / `builder.mutation` (e.g. `useGetNotesQuery`, `useAddNoteMutation`).  These hooks are imported where needed, such as in `src/sections/note/Notes.jsx`.
- The API slice is added to the Redux store in `src/store.js` (or via the Next.js `Provider` in `layout.jsx`).  This gives automatic cache invalidation, loading/error states and refetching capabilities.
- **Authentication** (if any) can be injected via `prepareHeaders` in `fetchBaseQuery` to attach a JWT or API key from `localStorage`/cookies.

## Development workflow

1. **Install dependencies**
   ```bash
   npm install
   ```
2. **Run the dev server**
   ```bash
   npm run dev
   ```
   The app is served at `http://localhost:3000`.
3. **API server** – make sure the backend defined by `NEXT_PUBLIC_API_URL` is running (e.g. `http://localhost:8000/api`).  The front‑end will automatically poll/mutate via the RTK‑Query hooks.
4. **Testing UI layout** – the project uses MUI components.  When using `Stack`, set `justifyContent` responsively (e.g. `justifyContent={{ xs: "flex-start", md: "space-between" }}`) to avoid flex‑box issues on small screens.
5. **Commit and PR process**
   - Follow the conventional‑commit format.
   - Run `npm run lint && npm run test` before pushing.
   - Open a PR against `master`; CI will run lint, type‑check and tests.
